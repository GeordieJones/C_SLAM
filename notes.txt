tracking_evaluator.h / .c (The Metrics Engine)

Robotics journals will not take your word that your tracking works; you must quantify it using standard Multi-Object Tracking (MOT) metrics. This library sits alongside your Scene Graph to monitor tracking quality, specifically focusing on handling occlusions and camera jitter.

    What it does: It tracks ground-truth object positions (either from a simulated dataset or manual validation frames) and compares them against your active Scene Graph IDs.

    Key Metrics to Compute:

        ID Switches (IDSW): The number of times a single physical object accidentally changes its tracking ID (e.g., when passing behind an obstacle). Minimizing this proves your object permanence logic works.

        Fragmentation (FRAG): How many times a track is interrupted and lost before being re-acquired.

        Localization Error: The precise Euclidean distance (in centimeters) between your tracked centroid and the true physical object center.



active_perception_analyzer.h / .c (The Map Convergence Tracker)

To publish on the concept that "some points do not need to be updated," you must mathematically prove that your selective voxel freezing algorithm saves processing power without degrading map accuracy.

    What it does: It acts as a telemetry layer over raycasting.c. Every time a ray is cast or a voxel state is modified, this library hooks in to profile the update density.

    Key Metrics to Compute:

        Voxel Update Skip Ratio: The percentage of voxels bypassed per frame because they were flagged as NODE_STATIC.

        Map Convergence Rate: How many frames it takes for an unfamiliar room to reach a stable state (99% of voxels unchanging) using your selective method versus a traditional "always-update" baseline.

        Voxel Divergence Error: A metric confirming that frozen regions didn't miss real structural changes (validating your Octree_invalidate_static_region threshold).

Gemini generate datasets that can be used to train:

1. For Autonomous/Mobile Robotics: The Waymo Open Dataset or KITTI

If your platform mimics a drone or a ground vehicle, use these industry gold standards:

    KITTI Vision Benchmark: A classic, highly manageable choice for a custom C loader. It provides synchronized camera images, calibration matrices (intrinsics/extrinsics), and 3D depth frames. It is lightweight enough to download and parse easily.

    Waymo Open Dataset: A vast, modern option containing millions of highly accurate 2D/3D frames, camera calibrations, and tracking IDs across distinct environmental conditions.

2. For General/Indoor Scenes: NYU Depth V2 or SUN RGB-D

If you are deploying an indoor robot, a robotic arm, or navigating standard rooms:

    NYU Depth V2: Contains thousands of video sequences of indoor scenes recorded by RGB and Depth cameras, paired with pre-baked ground truth depth data.

    SUN RGB-D: A comprehensive indoor benchmark providing spatial images with detailed 3D bounding box layouts.

3. For General Objects: Google Open Images V7

    If you just need your network to recognize standard everyday items, Open Images V7 houses over 16 million bounding boxes across 600 structural classes. It can be downloaded in partial fragments so you don't have to pull down the massive 500+ GB full set.

--------------METRICS--------------
Per function latency - microsecond time spent in each function - baseline against OPENCV
Jitter - standard variance in frame processing over a sequence of frames roughly(10,000) - < 2ms is good; >10 is bad
Peak & sustained memory allocation - total heap allocation tracked - 1 time memory allocations
frames per sec / watts - power consutption per throughput - 25FPS at 10W

voxel_skip ratio - computes saving of skipping un-active nodes - 60-80% in still enviorments
voxel_divergance error - distance between frozen map and ground truth - <2-5 cm
Map convergance rate - number of frames to get to stable map where nothing changes in still room- smooth curve

ID switches - times active tack changes id after moving behind something or other permance test - <5
Fragmentation - tracks number of times an object was lost then picked back up - 
centroid error - magnitude of distance from predicted to real

delta mean average precision - average precision change as model self learns
Quantization loss - loss from HAL optimizations
